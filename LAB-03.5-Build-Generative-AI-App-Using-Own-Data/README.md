# 📚 Azure AI Foundry Lab 3.5 – Build a Generative AI App Using Your Own Data

## Overview

This lab demonstrates how to build a Retrieval-Augmented Generation (RAG) application using Azure AI Foundry. A custom travel guide document was uploaded, converted into a searchable knowledge source, and connected to an AI agent capable of answering user queries using the provided data.

## Objectives

* Build a RAG-powered AI application using Azure AI Foundry.
* Create and configure a Knowledge Source and Knowledge Base.
* Generate vector embeddings for semantic retrieval.
* Develop an AI Travel Guide Agent using GPT-4o.
* Test retrieval-based responses using custom documents.

## Key Activities

* Deployed the GPT-4.1 model in Azure AI Foundry.
* Uploaded a custom travel guide document.
* Created a file-based Knowledge Source.
* Configured vector embeddings using **text-embedding-3-small**.
* Built and configured a Knowledge Base.
* Developed a Travel Guide Agent powered by GPT-4o.
* Connected the Knowledge Base to the agent.
* Tested the agent with travel-related prompts.

## Results

* Successfully implemented a Retrieval-Augmented Generation (RAG) workflow.
* Knowledge resources were created and integrated successfully.
* The AI agent generated accurate travel recommendations using the uploaded knowledge.
* Verified semantic retrieval and response generation within Azure AI Foundry.

## Technologies Used

* Azure AI Foundry
* GPT-4o
* GPT-4.1
* text-embedding-3-small
* Retrieval-Augmented Generation (RAG)
* Vector Embeddings
* Knowledge Sources
* Knowledge Bases

## Learning Outcomes

This lab provided practical experience in building RAG-based AI applications by integrating custom knowledge with Azure AI Foundry. It demonstrated the complete workflow from document ingestion and embedding generation to knowledge retrieval and intelligent response generation using an AI agent.
